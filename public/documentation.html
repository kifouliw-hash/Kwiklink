Documentation API KwikLink

L’API KwikLink vous permet d’intégrer des paiements instantanés — Mobile Money, cartes bancaires ou portefeuilles digitaux — directement dans vos applications. Nos endpoints sont sécurisés, rapides et compatibles avec les environnements sandbox et production.

Authentification

KwikLink utilise une clé API secrète pour authentifier vos requêtes :

Authorization: Bearer sk_test_abc123
Content-Type: application/json

Chaque requête à l’API doit contenir cet en-tête. Pour obtenir vos clés :

  • Connectez-vous à votre tableau de bord KwikLink
  • Accédez à l’onglet Clés API
  • Copiez votre clé de test ou de production

Créer un paiement

POST /v1/payment_intents
{
  "amount": 5000,
  "currency": "XOF",
  "method": "mobile_money",
  "customer": "client_001",
  "description": "Paiement marchand Abidjan"
}

Réponse :

{
  "status": "success",
  "transaction_id": "kl_39485GHQ",
  "redirect_url": "https://checkout.kwiklink.com/kl_39485GHQ"
}

Vérifier le statut d’un paiement

GET /v1/payments/{transaction_id}
{
  "status": "completed",
  "amount": 5000,
  "currency": "XOF",
  "method": "mobile_money"
}

Remboursement

POST /v1/refunds
{
  "transaction_id": "kl_39485GHQ",
  "amount": 5000
}

Réponse :

{
  "status": "refunded",
  "refund_id": "rf_20241019_001"
}

Webhooks

KwikLink vous notifie automatiquement lorsqu’un événement survient (paiement réussi, échec, remboursement…).

{
  "event": "payment.success",
  "data": {
    "transaction_id": "kl_39485GHQ",
    "amount": 5000,
    "currency": "XOF"
  }
}

SDKs & Intégrations

KwikLink propose des SDKs pour simplifier l’intégration :

// Node.js
const kwiklink = require('kwiklink')('sk_test_abc123');
await kwiklink.paymentIntents.create({ amount: 5000, currency: 'XOF' });

# Python
import kwiklink
kwiklink.api_key = "sk_test_abc123"
kwiklink.Payment.create(amount=5000, currency="XOF")
